## 빈 스코프
```
스프링 빈 스코프란?
스프링 빈은 기본적으로 싱글톤으로 하나의 객체만 생성되어 이를 공유 활용되는데
따라서 스프링 컨테이너는 빈 생성 및 관리를 해주며 이러한 빈들은 스프링 컨테이너와 생명주기를 함께한다

즉 스프링의 빈 스코프는 빈이 관리되는 범위를 뜻한다
```
## 싱글톤
```
default scope
스프링 컨테이너와 생명주기를 함께한다
```
## 프로토타입
```
프로토타입 스코프는 스프링 컨테이너가 빈의 생성과 의존관계 주입까지만 관여하고 더 이상 관리하지 않는 범위의 스코프.
싱글톤의 경우 빈을 호출하면 같은 빈을 돌려주지만 프로토타입은 호출마다 새로 만들어서 전달한다
즉 한 번 만들어진 프로토타입빈을 컨테이너로부터 다시 가져올 방법은 없다.

그렇기에 당연하게도 프로토타입 빈에서 싱글톤 빈을 호출하면 항상 같은 빈을 전달받으니 문제가 없지만
반대의 경우 프로토타입빈은 호출마다 새로운 빈을 반환해주니 문제가 생길 수 있다
```

## 그럼 프로토타입 스코프는 왜 사용되는가?
```
매 번 새로운 빈이 필요하다면?
싱글톤은 단일 객체를 공유하여 사용하는것인데 이를위해 불변을 도모한다.
즉 사용할 때 마다 상태가 변화되어야한다면 필요하다고 생각한다.
= 객체마다 상태 변화가 필요하다면 필요하다고 생각
```
